# Git Hub 와 파이참 프로젝트 연결 (master)

1. 깃허브에 리포지토리(저장소) 만들기함
=> url 복사

2. 파이참 실행 > 새 프로젝트 만들기 (리포지토리 이름과 프로젝트 명을 같게 하는것이 좋음)
3. 깃허브 저장소와 파이참 프로젝트 연결

git clone 복사한 url
git remote add origin https://github.com/사용자계정/프로젝트명.git
ex>
git remote add origin https://github.com/LukasBeanz/word_embed_project.git
연결 확인 :
git remote -v

표시되면 연결 성공:
origin  https://github.com/LukasBeanz/word_embed_project.git (fetch)
origin  https://github.com/LukasBeanz/word_embed_project.git (push)

4. 마스터가 프로젝트 구조에 대한 설정과 패키지 지정 등 파일 작업 진행함
# .gitignore 파일 작성 (깃허브에 푸시되면 안되는 파일들 설정)
# requirements.txt 작성
# README.md 작성
# 기본 앱 구조에 필요한 패키지 폴더, py 파일들 작성

5. 프로젝트 최초 커밋 (master)
브렌치 확인 :
git branch
처음 연결시에는
*master
마스터 브렌치이면 main 브렌치로 변경
git branch -M main
git branch
*main 확인

main 브렌치로 푸시함
git push -u origin main
=> 최초 푸시할 경우, 로그인 팝업 나타남 > 로그인하면 마스터 관리자 인증화면 나타남 > 탭 닫기 함
# 깃허브 브라우저에서 새로고침해서 업로드 확인

# 팀원들이 작업에 사용할 작업 브랜치를 깃 허브에서 만들어줌 (마스터)
=>리포지토리 : Code 탭에서 'main' 클릭 > 브랜치 이름 입력 > 바로 아래의 'create branch....' 클릭
git add .
git commit -m "Initial Commit"
